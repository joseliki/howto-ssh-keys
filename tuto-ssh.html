Añadir a github claves publicas generadas por nosotros mismos

</head> <body> <h2><b>crear claves privadas y publicas con ssh-keygen</b></h2> <p>para poder crear nuestro par de claves privada y publica lo haremos de la siguiente manera</p> <p>antes que nada, comentar un poco el funcionamiento de la identificacion y autenticacion por medio de claves privadas</p> <p>clave privada: Es utilizada por el cliente, para poder acceder al servidor ssh, para ello en el servidor</p> <p>debe existir una clave publica que fue generada junto con la clave privada</p> <p>dicho esto, explicaremos como se genera un par de claves privadas y publica</p> <ul> <li><b>1. preparacion: instalar el paquete ssh</b></li> <p>este paquete contendra la utilidad que necesitamos para poder crear el par de claves, la utilidad es ssh-keygen</p> <p>tambien contiene ssh-add que una vez creada la clave privada, esta se añadira al directorio .ssh del home del usuario, para no tener que ser ubicada con el parametro -i clave-privada<p> <li><b>2. creacion de la clave privada / publica</b></li> <p>llegados a este punto, deberiamos ser capaces de poder crear el par de claves sin problemas, para ello:</p> <p>con la utilidad ssh-keygen antes mencionada, ejecutaremos el siguiente comando como usuario sin privilegios:</p> <p>ssh-keygen -t rsa -C "tu-correo-electronico" -b 2048</p> <p>esto creara un par de claves privada / publica con tipo de algoritmo de encriptacion rsa, de 2048 bits de cifrado</p> <p>y como identificador tu correo electronico, sino se especifica el parametro -C creara la clave con el usuario@host de tu maquina</p> <li><b>a continuacion veremos algo como esto:</b><li> <p>Generating public/private rsa key pair.</p> <p>que nos indica que se esta generando..</p> <p>Enter file in which to save the key</p> <p>nos pregunta donde queremos guardar el par de claves, y que nombre quiere que le especifiquemos</p> <p>sino se especifica la ruta, la guardara en el directorio .ssh del home del usuario, y con el nombre id_rsa</p> <p>para la clave privada, y id_rsa.pub para la clave publica</p> <li><b>añadiendo la clave publica a github</b></li> <p>github necesita la clave publica para poder ser añadida, para ello, ejecutamos un cat /directorio/clave-publica</p> <p>copiamos su contenido, para añadirlo a github</p> <p>para poder añadir la clave publica, tenemos que editar nuestro perfil, dentro de el, pulsamos en la opcion ssh keys, y le damos a "add ssh key" para añadirla, le damos un nombre, y pegamos el contenido de la clave publica, en el area de contenido</p> <li><b>probando nuestra clave privada generada</b></li> <p>desde una terminal como usuario normal, escribimos el siguiente comando: ssh -T git@github.com</p> <p>nos debera aparecer algo como esto:</p> <p>primero nos avisa, de que la autenticidad con github no puede establecerse</p> <b>The authenticity of host 'github.com (192.30.252.128)' can't be established.</b> <p>despues nos muestra la clave rsa del servidor junto con su huella digital</p> <p><b>RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.</b></p> <p> y nos pregunta si estamos seguros de conectar</p> <p><b>Are you sure you want to continue connecting (yes/no)? yes</b></p> <p>aceptamos y se añadira a la lista de host conocidos la clave publica, que se guarda en el directorio .ssh/known_host del home del usuario</p> <p><b>Warning: Permanently added 'github.com,192.30.252.128' (RSA) to the list of known hosts.</b></p> <p>si todo funciono correctamente, deberemos ver algo como esto:</p> <p><b>Hi tu-usuario! You've successfully authenticated, but GitHub does not provide shell access.</b></p> <p>indicandonos que estamos correctamente autenticados, pero que github no provee acceso a shell</p> </ul> </body> </html>
